# 行为准则

OpenDocX 希望保持一个友好、直接、尊重事实的开源协作环境。

## 我们鼓励

- 用清晰、具体、可复现的方式描述问题。
- 对事不对人，围绕代码、体验和事实讨论。
- 接受不同使用场景下的优先级差异。
- 在提出批评时尽量给出改进建议。
- 贡献文档、截图、测试和真实案例。

## 我们不接受

- 人身攻击、羞辱、威胁或骚扰。
- 歧视性语言。
- 恶意刷屏、无关宣传或重复争吵。
- 在公开渠道披露他人隐私或安全漏洞细节。
- 故意提交含敏感信息、恶意代码或破坏性变更。

## 处理方式

维护者可以根据情况采取以下措施：

- 提醒或警告。
- 编辑或删除不合适内容。
- 关闭 Issue 或 PR。
- 临时或永久限制参与。

## 反馈

如果你遇到行为准则相关问题，请联系项目维护者。报告时请尽量提供上下文、链接和截图，方便判断与处理。
